# Presets

Les presets permettent de partir d’un **modèle de politique** plutôt que d’un robots.txt vide.

Pour beaucoup de sites WordPress, le preset est le point de départ le plus intelligent. Tu pourras affiner ensuite.

> **Pour la majorité des sites, commence par Essential.**

## Essential

**Idéal pour**
- la majorité des sites WordPress
- les sites vitrines / petites entreprises
- les propriétaires de site qui veulent un premier réglage sûr et pratique

**Pourquoi le choisir**
- Il améliore la configuration par défaut sans rendre la politique trop restrictive trop tôt.
- C’est le meilleur point de départ si tu n’es pas technique.
- Il te donne un parcours guidé avant de décider si des contrôles plus stricts sont réellement nécessaires.

**Moins adapté pour**
- les sites qui savent déjà qu’ils ont besoin de contrôles plus poussés sur l’IA ou l’archive
- les utilisateurs qui ont une vraie raison d’adopter une posture protection-first

## AI-First

**Idéal pour**
- les éditeurs
- les blogs et équipes éditoriales
- les sites de contenu qui veulent exprimer plus clairement leurs préférences d’usage IA

**Pourquoi le choisir**
- Il introduit une gouvernance plus profonde sur les IA et les LLM sans transformer le site en forteresse.
- C’est le bon choix quand tu veux distinguer plus clairement recherche, génération de réponse et entraînement.

**Moins adapté pour**
- les personnes qui n’ont besoin que d’un meilleur réglage par défaut
- les personnes qui n’ont pas besoin des contrôles payants

## Fortress

**Idéal pour**
- les sites orientés protection
- les équipes éditoriales plus strictes
- les sites qui veulent des contrôles plus forts sur l’archive, les bots et le scraping

**Pourquoi le choisir**
- C’est l’option guidée la plus restrictive du produit.
- Elle est utile quand l’ouverture n’est pas ta priorité.
- Elle correspond aux cas où l’archive, le scraping ou une plus large exposition aux bots sont de vraies préoccupations.

**Moins adapté pour**
- les débutants qui veulent simplement une configuration propre par défaut
- les sites qui bénéficient d’un modèle de découverte plus ouvert

## Custom

**Idéal pour**
- les utilisateurs avancés
- les agences
- les personnes qui veulent construire la politique module par module

**Pourquoi le choisir**
- Il offre plus de liberté quand tu comprends déjà les compromis.
- C’est la bonne option si ton site ne rentre pas proprement dans un profil de preset standard.

**Moins adapté pour**
- les utilisateurs qui découvrent encore l’impact des différentes catégories de crawlers
- les personnes qui veulent le point de départ le plus rapide et le plus sûr

## Presets par type de site

| Type de site | Preset recommandé |
| --- | --- |
| Site vitrine / petite entreprise | Essential |
| Site de contenu / éditeur | AI-First |
| Boutique WooCommerce | Essential ou AI-First |
| Site sensible à l’archive ou orienté protection | Fortress |
| Agence / utilisateur avancé | Custom |

## Erreurs fréquentes

- Choisir Fortress trop tôt sans comprendre les compromis
- Attendre d’un preset qu’il agisse comme un pare-feu
- Surbloquer les bots sociaux ou de découverte sans raison claire
- Commencer par Custom avant de comprendre la logique du produit
